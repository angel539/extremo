metaRDF

An extensible recommendation system for meta-modelling

From Angel Mora-Segura at April, 2016

Meta-models play a pivotal role in Model-Driven Engineering (MDE). They are used to create domain-specific models, and to create model management operations like model transformations or code generators. However, meta-modelling is currently mostly a manual activity, which does not profit from existing knowledge.

In order to facilitate the meta-modelling task, in this paper we present an extensible recommendation system for meta-modelling. This recommendation system permits the provision of heterogeneous data description sources (like Ontologies, XML Schemas, Data-Bases schemas and meta-models), and enables their uniform querying. Different kinds of queries are supported, and improved through synonym search. The results of queries are prioritized trhough sense disambiguation, and can be visualized graphically.

The recommender architecture has been designed to be independent of the meta-modelling technology used. As a proof of concept, we show its integration within DSL-tao, a pattern-based meta-modelling tool. The usefulness of the system is illustrated with a running example in the process modelling domain.

Class diagram

#1_ Defining your own format assistant

Eclipse > New > Other > Plugin project > ... > Finish

In MANIFEST.MF insert a Dependency to metaRDF.core.extensions and metaRDF.core. Both will give you access to the core classes to implement.

Add an extension to metardf.core.extensions.assistant and then a new Assistant. Define a familiar name (this will be used later to show a pointer from file resources and assistant by the UI), a class name, and file extensions this assistant will be applied.

A new all-methods-empty class was created implementing the IFormatAssistant class. This class needs to parse your format to our class structure (shown in the class diagram above).

#2_ Extending the core model

Create a new package in your plugin and call it something like "... .model". There will be the place where we will extend the core classes of the plugin. The classes to be overwritten are:

  • metardf.core.model.impl.SemanticClass
  • metardf.core.model.impl.ObjectProperty
  • metardf.core.model.impl.DataProperty

In order to give metaRDF information about the core classes have been extended they must be registered through the getRegisteredTypes() method in the assistant


With the implementation 3 assistants are given to cover three of the main data entities format representation:

  • OWL
  • RDF
  • Ecore
all of them implements the method getClassesLike(Map namesByRelevance) which receives a map composed by the strings suggested in the search and a weight value representing certain relevance to order the words. The weights are preset by the tool but they can be overwritten in this point calling the method setWeight(...) in your extended version of SemanticClass.
